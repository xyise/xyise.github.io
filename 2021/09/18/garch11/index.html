Opening

In financial time series data, it is common to observe volatility clusturing, meanining large changes tend to be clustered. For example, the figure below shows daily returns (light blues) of the S&P500 index where one can observe clusters of larges changes during 2009 (financial crisis) and 2020 (Covid-19 stress).

In this document, we demonstrate how the volatility clustering phenomena can be analysed using Garch(1,1), one of the model simplest models featuring heteroskedasticity (a fancy word indicating that the volatility is varying over time as indicated by the red line in the above figure.)

Garch(1,1) Model

The model describes the dynamics for return variable rtr_trt​ and volatility variable σt\sigma_tσt​:
\begin{eqnarray}
r_t&=&\mu+\sigma_t\epsilon_t \label{E:garch-return}\
\sigma_{t+1}^2 & = &\sigma_\infty^2(1 - \beta-\gamma) + \beta \sigma_{t}^2 + \gamma \sigma_{t}2\epsilon_{t}2 \label{E:garch-vol}\
\end{eqnarray}
where

  • ϵt\epsilon_tϵt​ is the innovation variable, assumed to be Gaussian (for simplicity) and i.i.d. over time.
  • μ\muμ is the drift parameter for rtr_trt​
  • σ∞\sigma_\inftyσ∞​, β\betaβ and γ\gammaγ are Garch(1,1) model parameters with
    σ∞>0,β≥0,γ≥0,β+γ<1.\sigma_\infty > 0,\quad \beta \ge 0, \quad \gamma \ge 0,\quad \beta + \gamma < 1.σ∞​>0,β≥0,γ≥0,β+γ<1.

References: See this wiki page for the model general model Garch(p,q) as well as other Garch families.

Properties

The followings are some of notable properties of the model:

  • Re-writing the second equation above, it is easy to see that the Garch volatility σt\sigma_tσt​ is mean-reverting:
    \begin{equation}
    \sigma_{t+1}^2 - \sigma_{t}^2 = (1 -\beta - \gamma)(\sigma_{\infty}^2 - \sigma_{t}^2) + \gamma\sigma_{t}2(\epsilon_{t}2-1)
    \end{equation}
    where σt\sigma_tσt​ is the mean-reverting level and (1−β−γ)(1-\beta-\gamma)(1−β−γ) is the mean-reverting speed.
  • Given the information up to today ttt, the expected forward variance at τ>t\tau > tτ>t is fully determined and calculated as
    \begin{eqnarray*}
    \bar{\sigma}^2_t(\tau) & := & E[\sigma_{t+\tau}^2| \mathcal F_t ]\
    & = & \sigma_{\infty}^2 + (\beta+\gamma){\tau-1}(\sigma_{t+1}2 - \sigma_\infty^2)
    \end{eqnarray*}

Garch Filtering

Denoting the index price time series as
S0,S1,⋯ ,St,⋯ ,S_0, S_1, \cdots, S_t, \cdots,S0​,S1​,⋯,St​,⋯,
the goal is to derive the Garch volatility time series (red line in the above figure)
σ0,σ1,⋯ ,σt,⋯ .\sigma_0, \sigma_1, \cdots, \sigma_t, \cdots.σ0​,σ1​,⋯,σt​,⋯.

To this end,

  • Set rtr_trt​ is the log-change of StS_{t}St​, i.e.
    rt=ln⁡St/St−1.r_t = \ln { S_t / S_{t-1}}.rt​=lnSt​/St−1​.
  • μ\muμ and σ∞\sigma_\inftyσ∞​ are set to the unconditional mean and standard deviation of {rt}{r_t}{rt​} over all available data, respectively.
  • For now, we set β=0.9\beta = 0.9β=0.9 and γ=0.09\gamma = 0.09γ=0.09, which would be within typical ranges (based on google search). The estimation method is discussed later this document.

Then, σt\sigma_tσt​ can be derived using the following simple recursive algorithm:

  • Initialise: σ02=σ∞2\sigma^2_0 = \sigma^2_\inftyσ02​=σ∞2​ (or other value that is defendable)
  • Assume that we have the volatility σt\sigma_tσt​ as of time ttt. Then,
    1. Set the innovation term ϵt=(rt−μ)/σt\epsilon_t = (r_t - \mu) / \sigma_tϵt​=(rt​−μ)/σt​, which is from ().
    2. Set the next volatility σt+1\sigma_{t+1}σt+1​ using ().

The above algorithm produces the corresponding innovation time series (illustrated figure below)
ϵ1,⋯ ,ϵt,⋯ ,\epsilon_1, \cdots, \epsilon_t, \cdots,ϵ1​,⋯,ϵt​,⋯,
which are daily returns with Garch volatilities removed.

While one may run many statistical tests, it is visually obvious that the filtered returns (ϵt\epsilon_tϵt​) are more i.i.d. than the original returns (rtr_trt​). So, from the statistically perspective, the former samples are more meaningful to work with.

Estimation: MLE

So far, we have postulated certain values for β\betaβ and γ\gammaγ without justifications. In this section, we estimate them by applying the standard maximum likelihood estimation method.

Objective Function

To this end, noting that σt\sigma_tσt​ is determined one time step in advance:
rt∣Ft−1∼N(μ,σt2),r_t | \mathcal F_{t-1} \sim \mathcal N (\mu, \sigma_{t}^2),rt​∣Ft−1​∼N(μ,σt2​),
the likelihood function for time step ttt is given as
\begin{equation}
l(r_t|\mathcal F_{t-1}) = \displaystyle \frac{1}{\sqrt{2\pi \sigma_t^2}}
\exp\left( - \frac{(r_t - \mu)2}{2\sigma_t2}\right).
\end{equation}
Thus, we can write the objective function to maximize as below after doing the usual steps (multiply over samples, take log to turn into sum, ignore constant factors)
\begin{equation}
L(\beta, \gamma) = -\sum_{t} \left[ \ln \sigma_t^2 + \frac{(r_t - \mu)2}{\sigma_t2}\right].
\end{equation}

Optimisation

Before running the actual optimisation function, we calculate LLL over a range of β\betaβ and γ\gammaγ values with constraint β+γ<1\beta + \gamma < 1β+γ<1 and obtain the following contour plot which indicates where the maximum is roughly located.

Running an optimisation routine (e.g. Nelder-Mead), the maximum is achieved at
(β,γ)=(0.88035162,0.10633708),(\beta, \gamma) = (0.88035162, 0.10633708),(β,γ)=(0.88035162,0.10633708),
which is not too far from our postulated values (0.9, 0.09).

In Closing

which can be easily shown to be the EWMA volatility. For typical parametrisations (e.g. \beta = 0.9), EWMA volatilities are very close to Garch(1,1) volatilities. But, as a long-term forecasting model, EWMA is not suitable (try it out).
